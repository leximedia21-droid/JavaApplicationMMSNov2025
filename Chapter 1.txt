1.7 Explain the two compilation phases of Java programs. 
ANS: Phase 1: Compilation to bytecode
- The Java compiler javac takes your .java source code and compiles it into platform-independent bytecode.
- This produces .class files. Bytecode isn’t machine code for your CPU. It’s an intermediate code for the Java Virtual Machine.
- At this stage syntax is checked, types are verified, and optimizations may be done.
Phase 2: Interpretation/JIT compilation to native code
- The JVM loads the .class bytecode and executes it. 
- Initially the JVM interprets bytecode line by line. For code that runs often, the JIT (Just-In-Time) compiler converts hot bytecode into native machine code for your CPU.
- This is why Java is "write once, run anywhere". Same bytecode runs on Windows, Mac, Linux as long as there’s a JVM.

1.8 One of the world’s most common objects is a wrist watch. Discuss how each of the following terms and concepts applies to the notion of a watch: object, attributes, behaviors, class, inheritance (consider, for example, an alarm clock), modeling, messages, encapsulation, interface and information hiding.
ANS: Using a wrist watch to explain core OOP terms:

Object: A specific, real instance. Your actual Casio G-Shock on your wrist right now is an object. It exists in memory with its own state.

Class: The blueprint for making watches. The Watch class defines what all watches have and do. Your G-Shock and my Apple Watch are both objects created from the Watch class.

Attributes: Data that describes the object’s state. For a watch: time = 11:05 AM, color = black, brand = "Casio", is Waterproof = true, alarm time = 7:00 AM.

Behaviors: Things the watch can do. Methods like display time(), set alarm(), start stopwatch(), beep().

Inheritance: Building specialized watches from general ones. “Alarm Clock Watch” inherits from Watch. It gets all basic watch behavior for free, then adds snooze() and alarm ring(). Both digital watch and analog watch could inherit from Watch.

Modeling: Creating a simplified software version of a real watch. We don’t model every gear and spring. We only model attributes and behaviors that matter to our program.

Messages: Objects talk by sending messages. You press a button = sending a set time (11, 05) message to the watch object. The watch responds by changing its state.

Encapsulation: The watch hides its internal complexity. You don’t need to know about quartz crystals or gears to use it. The internal battery level and circuit logic are private. You only access it through public buttons/methods.

Interface: The contract of what you can do. The watch interface promises show time (), set alarm(time), toggle24HourMode (). Smart watch might implement Watch interface plus touch screen interface.

Information hiding: A part of encapsulation. The watch hides how it keeps time internally.

1.10 (Test-Drive: Body Mass Index Calculator) 
Obesity causes significant increases in illnesses such as diabetes and heart disease. To determine whether a person is overweight or obese, you can use a measure called the body mass index (BMI). The United States Department of Health and Human Services provides a BMI calculator at http://www.nhlbi.nih.gov/guidelines/obesity/BMI/ bmicalc.htm. Use it to calculate your own BMI. A forthcoming exercise will ask you to program your own BMI calculator. To prepare for this, use the web to research the formulas for calculating BMI. 
ANS: Core BMI formulas:
- Metric: BMI = \frac{weight\ in\ kg}{(height\ in\ m)^2}
- Imperial: BMI = \frac{weight\ in\ lb}{(height\ in\ in)^2} \times 703
Standard BMI categories from NIH/WHO:
Category	BMI Range
Underweight	< 18.5
Normal weight	18.5 - 24.9
Overweight	25.0 - 29.9
Obese	30.0+
Example: 
If you weigh 70 kg and are 1.75 m tall: 
BMI = \frac{70}{1.75^2} = \frac{70}{3.0625} = 22.9. That's "Normal weight".

1.12 (Gender Neutrality) 
Many people want to eliminate sexism in all forms of communication. You’ve been asked to create a program that can process a paragraph of text and replace gender-specific words with gender-neutral ones. Assuming that you’ve been given a list of gender-specific words and their gender-neutral replacements (e.g., replace both “wife” and “husband” with “spouse,” “man” and “woman” with “person,” “daughter” and “son” with “child”), explain the procedure you’d use to read through a paragraph of text and manually perform these replacements. How might your procedure generate a strange term like “woperchild?” You’ll soon learn that a more formal term for “procedure” is “algorithm,” and that an algorithm specifies the steps to be performed and the order in which to perform them. We’ll show how to develop algorithms then convert them to Java programs which can be run on computers.
ANS : Procedure / Algorithm: Given a list of gender-specific words and their neutral replacements:
1. Input: Take the paragraph of text and the dictionary of replacements. Example: {"man": "person", "woman": "person", "son": "child", "daughter": "child"}
2. Tokenize: Split the paragraph into individual words. Handle punctuation separately so "woman." becomes ["woman", "."].
3. Scan each word: For each word in the paragraph:
   - Convert to lowercase for comparison, but preserve original case
   - Check if the word matches any key in your replacement list
   - If match found, replace it with the gender-neutral term
   - If no match, keep the original word.
4. Rebuild: Join the words back into a paragraph with proper spacing and punctuation
5.. Output: Return the modified paragraph

How you might get "woperchild"
This bug happens if your algorithm does a naive substring replacement instead of whole-word replacement.
Example: 
Original text: "The woman ran to her son."  
Replacement list: {"man": "person", "son": "child"}
